工程變更最麻煩的，往往不是計價方法，而是幾個月後沒有人說得清：誰在何時指示、現場做了多少、設計何時落實，以及有沒有影響工期。

## Variation register 最少應有甚麼？

| 欄位 | 用途 |
| --- | --- |
| 編號 | 建立單一 reference |
| 指示日期及來源 | 追蹤正式／非正式指示 |
| 變更描述 | 說明範圍，不只寫「見附件」 |
| 初步估值 | 早期成本預警 |
| 報價及回覆日期 | 追蹤商務流程 |
| 進度影響 | 連結 programme 和 EOT 評估 |
| 狀態 | Open / Submitted / Agreed / Closed |

## 不要等正式 VO 才開紀錄

現場口頭要求、圖紙修訂、RFI 回覆，都可能帶來變更。先記錄「潛在變更」，再由合約團隊判斷性質，比事後重組證據可靠。

## 每週更新，而不是月尾補數

一張半年沒有更新的 register，只是歷史檔案。真正有管理價值的 register 應該在項目會議前更新，並突出：

- 金額大但未議定的項目；
- 已施工但文件未齊的項目；
- 可能影響關鍵路徑的項目；
- 長期等待對方決定的項目。
